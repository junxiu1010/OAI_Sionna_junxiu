# 새 프로젝트 폴더 구성 가이드

## 개요

이 문서는 `DevChannelProxyJIN/`이라는 최상위 폴더에서 **OAI(openairinterface5g_whan)**, **Sionna Channel Proxy(vRAN_Socket)**, 그리고 **NVIDIA Aerial cuPHY(aerial-git)**를 통합 관리하는 구조를 설명한다.

- **OAI**: 호스트에서 직접 실행 (CPU 기반, L2/MAC)
- **Sionna Proxy**: Docker 컨테이너에서 실행 (GPU 기반, 채널 시뮬레이션)
- **Aerial cuPHY**: Docker 컨테이너에서 실행 (GPU 기반, L1/PHY)
- **Cursor AI**: 호스트에서 모든 코드를 동시에 편집/분석

│       └── ...

├── openairinterface5g_whan/             ← 호스트에서 직접 실행 (CPU)

## 컨테이너 구분

| 폴더 | 컨테이너 여부 | 실행 환경 | 연산 장치 | 역할 |
|------|:---:|------|:---:|------|
| `DevChannelProxyJIN/` | ❌ 아님 | 호스트의 일반 폴더 | - | 프로젝트 루트 |

**컨테이너는 2개다**: Sionna Proxy (`oai_sionna_proxy`)와 Aerial cuPHY (`c_aerial_$USER`).

## 작업 방식

### 방식 A: 호스트에서 열기 (추천)

- Cursor SSH 접속 → `DevChannelProxyJIN/` 폴더 열기
- Cursor 한 창에서 OAI + Sionna + Aerial 코드 편집 **전부 가능**
- OAI 빌드/실행은 호스트 터미널에서 직접
- Sionna 실행은 `docker exec -it oai_sionna_proxy ...`
- Aerial 실행은 `docker exec -it c_aerial_$USER ...`

**장점**: 한 창에서 편집, 빌드, 실행 전부 가능
**단점**: Python IntelliSense(자동완성) 안 됨

### 방식 B: Dev Containers로 컨테이너에 접속

- Cursor SSH 접속 → 컨테이너 실행 → Dev Containers: Attach → 해당 경로 열기
- **단, 다른 컨테이너와 호스트의 OAI 빌드/실행은 별도 터미널이 필요**

> **한 창에서 편집과 실행을 모두 하려면 방식 A를 사용한다.**

## 구성파일의 정체와 역할

### ① `DevChannelProxyJIN.code-workspace`

- **정체**: Cursor(VS Code) 멀티루트 워크스페이스 설정 파일 (JSON 형식)
- **위치**: `DevChannelProxyJIN/` 루트
- **역할**: 여러 폴더를 하나의 Cursor 창에서 동시에 열어준다.
- **필요한가?**: `DevChannelProxyJIN/` 폴더 자체를 열면 전부 보이므로, **일반적으로 안 써도 된다.**

### ② `docker-compose.yml` (루트)

- **정체**: Docker Compose 설정 파일 (YAML 형식)
- **위치**: `DevChannelProxyJIN/` 루트
- **역할**: **Sionna 컨테이너**를 어떤 조건으로 띄울지 정의한다.
- **핵심**: `vRAN_Socket/`과 `openairinterface5g_whan/` 모두 컨테이너 안 `/workspace`에 마운트한다.

### ③ `vRAN_Socket/Dockerfile`

- **정체**: Sionna 컨테이너 이미지 빌드 스크립트
- **위치**: `vRAN_Socket/` 안
- **역할**: Sionna 컨테이너에 설치할 것들을 정의한다 (Python, TensorFlow, CuPy, CUDA 등).

### ④ `cuPHY-CP/container/run_aerial.sh` (Aerial용)

- **정체**: Aerial 컨테이너 실행 스크립트
- **위치**: `aerial-git/aerial-cuda-accelerated-ran/cuPHY-CP/container/`
- **역할**: NGC 이미지 기반으로 Aerial 개발 컨테이너를 실행한다.
- **핵심**: 호스트 소스를 `/opt/nvidia/cuBB`에 마운트, GPU/hugepages/GDRCopy/shared memory 자동 설정.

### ⑤ `docker/docker-compose.oai-cuphy-cp.yaml` (Aerial OAI 연동용)

- **정체**: Docker Compose 설정 파일
- **위치**: `aerial-git/aerial-cuda-accelerated-ran/docker/`
- **역할**: OAI gNB + cuPHY-CP + (Sionna) + (UE)를 한 번에 올리는 통합 구성.
- **비고**: 별도 서버 간 연동이나 전체 E2E 테스트 시 사용.

## 실행 흐름

### 흐름 A: 기존 소켓 방식 (G0, Sionna Proxy)

1. Cursor에서 Remote SSH로 서버 접속
2. `DevChannelProxyJIN/` 폴더 열기
3. `docker compose up -d oai_sionna_proxy` → Sionna 컨테이너 실행
4. Cursor 터미널을 여러 개 띄워서:
   - 터미널 1: gNB 실행 (`sudo ./nr-softmodem ...`)
   - 터미널 2: Sionna 실행 (`docker exec -it oai_sionna_proxy python3 ...`)
   - 터미널 3: UE 실행 (`sudo ./nr-uesoftmodem ...`)
5. gNB ↔ Sionna Proxy ↔ UE 가 **TCP 소켓**(port 6017, 6018)으로 통신

### 흐름 B: Aerial 방식 (GPU L1)

1. Cursor에서 Remote SSH로 서버 접속
2. `DevChannelProxyJIN/` 폴더 열기
3. Aerial 컨테이너 실행:
   ```
   cd aerial-git/aerial-cuda-accelerated-ran/cuPHY-CP/container
   ./run_aerial.sh -d
   ```
4. 컨테이너 안에서 SDK 빌드 (최초 1회):
   ```
   docker exec -it c_aerial_$USER bash
   ./testBenches/phase4_test_scripts/build_aerial_sdk.sh
   ```
5. cuPHY L1 서버 실행:
   ```
   docker exec -it c_aerial_$USER bash -c \
     './build.x86_64/cuPHY-CP/cuphycontroller/examples/cuphycontroller_scf -c <config>'
   ```
6. OAI gNB를 FAPI 모드로 실행 (호스트):
   ```
   export FAPI_AERIAL_ENABLE=1
   sudo -E ./nr-softmodem -O <conf> --rfsim ...
   ```
7. OAI gNB(L2) ↔ Aerial cuPHY(L1)가 **FAPI(UDP)**로 통신

## 통신 구조

### 기존: 소켓 방식 (G0)

```
호스트                              oai_sionna_proxy 컨테이너
┌──────────┐                    ┌──────────────────┐
│   gNB    │ ── port 6017 ───→ │  Sionna Channel  │
│  (L2+L1) │                    │     Proxy        │
│   UE     │ ← port 6018 ──── │  (채널 시뮬레이션) │
└──────────┘                    └──────────────────┘
  (CPU)                            (GPU, Docker)

OAI가 L1+L2 모두 처리, Sionna는 IQ 레벨 채널만 시뮬레이션
```

### Aerial 방식 (GPU L1)

```
호스트                      aerial 컨테이너         oai_sionna_proxy 컨테이너 (선택)
┌──────────┐            ┌──────────────────┐    ┌──────────────────┐
│  OAI gNB │ ──FAPI──→ │  cuPHY (L1/GPU)  │    │  Sionna Channel  │
│  (L2/MAC)│  (UDP)     │  LDPC,FFT,MIMO   │──→ │  (채널 시뮬레이션) │
│          │            │  cuphycontroller │    │                  │
└──────────┘            └──────────────────┘    └──────────────────┘
  (CPU)                    (GPU, Docker)           (GPU, Docker)

OAI는 L2(MAC)만 처리, L1(PHY)은 전부 GPU에서 cuPHY가 수행
```
