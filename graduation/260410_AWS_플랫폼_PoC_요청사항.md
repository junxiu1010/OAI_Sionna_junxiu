# AWS 연구 환경 플랫폼 PoC 요청사항

---

## 1. 연구 주제

**6G Extreme Massive MIMO에서의 AI 기반 채널 공분산 추정 및 고유빔 프리코딩 최적화**

> AI-driven Channel Covariance Estimation and Eigenbeam Precoding Optimization  
> for Extreme Massive MIMO in 6G FR3

### 핵심 키워드

Extreme Massive MIMO (1024 AE), Weichselberger channel model, CSI acquisition,  
covariance estimation, eigenbeam precoding, UE scheduling, AI/ML,  
3GPP NR FR3 (7.5 GHz), 38.901 channel model

### 연구 범위

- 1024-AE급 기지국($32 \times 32$ 패널)에서의 Weichselberger 채널 모델 기반 CSI 획득 및 채널 공분산 추정
- Kronecker 구조를 넘어선 jointly-correlated 채널의 커플링 행렬($\Omega$) 및 marginal covariance($R_{\text{BS}}$, $R_{\text{UE}}$) 학습
- 추정된 공분산의 고유구조를 활용한 프리코딩 및 스케줄링 최적화
- AI/ML을 통한 채널 행렬 → 공분산 추정 가속, 프리코더 직접 출력

### 논문 수집 참고

- 위 키워드 기반 SCI/KCI 논문 40~60편 수집 요청 (학교 계정 별도 전달)

## 3. 학습 데이터

자체 파이프라인으로 원시 데이터(Ray, 채널 H)를 대규모 생성 가능하며, 두 종류를 제공합니다.

### 3-1. Ray 데이터 — Sionna RT 레이 트레이싱 출력

3D 도시 모델 기반 물리적 전파 경로 정보입니다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `tau` | float32 | 경로 지연 (초) |
| `power` | float32 | 경로 전력 |
| `theta_r_deg`, `phi_r_deg` | float32 | 수신측 도래각 (ZoA, AoA) |
| `theta_t_deg`, `phi_t_deg` | float32 | 송신측 출발각 (ZoD, AoD) |
| `los_nlos_flag` | int32 | LoS / NLoS 구분 |

- **형식**: RX별 `.npy` 파일 (e.g., `Area1_7.5GHz_Ray_RX42_tau.npy`)
- **규모**: 에어리어당 수백~수천 RX, 다중 에어리어 × 다중 주파수

### 3-2. 채널 행렬 $H$ — 38.901 절차 기반 AE OFDM 채널

Ray 데이터로부터 3GPP 38.901 절차에 따라 생성한 Antenna Element OFDM 채널입니다.

| 항목 | 값 |
|------|-----|
| **채널 크기** | $H \in \mathbb{C}^{16 \times 1024 \times 128}$ (RX AE × TX AE × 서브캐리어) |
| **안테나 구성** | TX: $32 \times 32 = 1024$ AE, RX: $4 \times 4 = 16$ AE |
| **OFDM** | FFT size 128, SCS 120 kHz |
| **Realization** | RX당 256회 독립 정적 채널 (매회 새로운 초기 위상) |
| **데이터 타입** | complex64 (8 bytes/element) |

- **RX당 데이터량**: 약 4.1 GB (256 realization × 16 × 1024 × 128 × 8 bytes)
- 전체 에어리어 수천 RX에 대해 선택적 저장 가능 (realization 수 조절)

### 데이터 생성 역량

- **P1A**(레이 트레이싱) → **P1B**(유효 RX 필터) → **P1F**(채널 H 생성) 자동화 파이프라인 완성
- 다른 서버에서 이미 수집 중인 데이터 보유, 기존 결과물 즉시 활용 가능
- 파라미터 변경만으로 신규 시나리오(에어리어, 주파수, 안테나 구성) 확장 가능
- AWS 플랫폼 위에서도 동일 파이프라인 구동 → 생성 → 학습 → 평가 사이클 일괄 운용 가능
